4. CLICK SOUND SYSTEM
   - Implementasi fungsi click_sound yang sudah dikonfigurasi di gui.yml
   - Setiap item di GUI kini memiliki audio feedback saat diklik
   - 21 item dengan sound yang dikustomisasi
   - Dynamic sound berdasarkan state (active/inactive, ada/kosong)
   
   File yang dimodifikasi:
   - MainMenuGUI.java: 
     * Method playClickSound(Player, int slot)
     * Method getClickSoundPath(int slot) - mapping slot ke config path
     * Method playQuickActionsSound(Player, Material) - untuk Quick Actions menu
   - GUIListener.java:
     * Integrasi playClickSound() di handleMainMenuGUI()
     * Integrasi playQuickActionsSound() di handleQuickActionsGUI()
   
   Sound Configuration Examples:
   - Player Head: "entity.player.levelup"
   - President (ada): "block.anvil.use"
   - President (kosong): "block.anvil.break"
   - Arena (active): "entity.player.attack.crit"
   - Arena (inactive): "entity.player.attack.sweep"
   - Recall (active): "block.redstone_torch.burnout"
   - Recall (inactive): "block.torch.extinguish"
   - Treasury: "block.note_block.pling"
   - Election: "item.book.page_turn"
   
   Features:
   - Fallback sound (UI_BUTTON_CLICK) jika sound tidak valid
   - Sound dimainkan SEBELUM action (instant feedback)
   - Volume 1.0f, Pitch 1.0f (dapat disesuaikan)

6. FLEXIBLE GUI ACTION SYSTEM
   - Sistem GUI yang lebih fleksibel dengan konfigurasi on_click di gui.yml
   - Setiap item kini dapat dikustomisasi untuk membuka GUI berbeda
   - Administrator dapat mengubah fungsi item tanpa perlu edit code Java
   - Mendukung multiple states (default/no_president, active/inactive)
   
   File yang ditambahkan:
   - GUIAction.java (NEW):
     * Enum untuk semua possible GUI actions
     * Method fromConfig(String) untuk parsing action dari config
     * 18 predefined actions (11 OPEN_GUI_*, 7 ACTION_*)
   
   File yang dimodifikasi:
   - gui.yml:
     * Menambahkan on_click untuk semua 24 items
     * Menghapus gui_id (tidak digunakan di code)
     * Header dokumentasi komprehensif (47 baris)
     * Daftar lengkap semua available actions
   - MainMenuGUI.java:
     * Method getGUIAction(String itemKey, String state)
     * Method getGUIAction(String itemKey) - overload tanpa state
     * Support untuk reading on_click dari config dengan state support
   
   Available Actions:
   
   === OPEN GUI ACTIONS (11) ===
   - <open_gui_main_menu>          : Opens Main Menu GUI
   - <open_gui_player_stats>       : Opens Player Statistics GUI
   - <open_gui_government>         : Opens Government GUI
   - <open_gui_voting>             : Opens Voting/Election GUI
   - <open_gui_cabinet>            : Opens Cabinet GUI
   - <open_gui_treasury>           : Opens Treasury GUI
   - <open_gui_executive_orders>   : Opens Executive Orders GUI
   - <open_gui_recall>             : Opens Recall System GUI
   - <open_gui_history>            : Opens Presidential History GUI
   - <open_gui_leaderboard>        : Opens Leaderboard GUI
   - <open_gui_help>               : Opens Help/Guide GUI
   
   === SPECIAL ACTIONS (7) ===
   - <action_register_candidate>   : Register as presidential candidate
   - <action_rate_president>       : Show rating command message
   - <action_close_inventory>      : Close inventory
   - <action_arena_info>           : Show arena information
   - <action_endorse_candidate>    : Show endorse command message
   - <action_donate_treasury>      : Show donate command message
   - <action_join_arena>           : Join presidential arena
   
   Configuration Examples:
   
   # Single state item
   player_head:
     gui_slot: 4
     material: PLAYER_HEAD
     on_click: "<open_gui_player_stats>"
   
   # Multiple states item
   president_item:
     gui_slot: 10
     default:  # When president exists
       material: GOLDEN_HELMET
       on_click: "<open_gui_government>"
     no_president:  # When no president
       material: BARRIER
       on_click: "<open_gui_voting>"
   
   Features:
   - Case-insensitive action parsing
   - Support untuk angle brackets format <action>
   - Fallback ke UNKNOWN action jika tidak ditemukan
   - Multiple state support (default, no_president, active, inactive)
   - Backward compatible dengan sistem click handling lama
   - Hot-reload supported
   
   Benefits:
   - Fleksibilitas: Ubah fungsi item tanpa edit Java
   - Maintainability: Semua config GUI di satu tempat
   - Customization: Layout dan behavior dapat disesuaikan server
   - Extensibility: Mudah menambah GUI/action baru
   
   Dokumentasi:
   - .docs/GUI-FLEXIBLE-SYSTEM.md (NEW):
     * Panduan lengkap sistem GUI fleksibel
     * Struktur konfigurasi dan format
     * Daftar semua actions dengan deskripsi
     * Cara menambahkan GUI/action baru
     * Contoh implementasi
     * Troubleshooting guide
     * Future enhancements roadmap
