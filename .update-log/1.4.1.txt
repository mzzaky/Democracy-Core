[PERBAIKAN BUG]
- Fix: Command `/dc admin set president <player>` selalu mengembalikan
  "player not found" meskipun player sedang online
  → Root cause: `Bukkit.getOfflinePlayer()` di 1.20+ selalu mengembalikan
    objek OfflinePlayer (dengan offline UUID) bahkan untuk player yang
    belum pernah bermain, sehingga `hasPlayedBefore()` selalu false
  → Fix: Mengubah logika pencarian: pertama coba `Bukkit.getPlayer()`
    untuk player online, baru fallback ke `getOfflinePlayer()` dengan
    pengecekan `hasPlayedBefore()` untuk player offline

- Fix: Mengganti Material kustom di gui.yml tidak berpengaruh (GUI tetap
  menampilkan material lama / item tidak bisa diklik)
  → Root cause #1: GUIListener memeriksa tipe material secara hardcode
    (misal: clicked.getType() == Material.LECTERN) — jika material
    diganti di config, klik tidak akan pernah ter-trigger
  → Root cause #2: Metode createCabinetItem, createTreasuryItem,
    createElectionItem, createArenaItem, createHistoryItem,
    createMyStatsItem, createLeaderboardItem, createHelpItem, dan
    createExecutiveOrdersItem membuat item dengan Material hardcode,
    bukan membaca dari gui.yml
  → Root cause #3: playClickSound() menggunakan nomor slot hardcode
    (case 4, 10, 12...) — jika gui_slot diubah di config, sound
    tidak terpicu

  → Fix GUIListener: semua click handler di handleMainMenuGUI()
    sekarang hanya mengecek SLOT (dari config), tidak lagi mengecek
    tipe material
  → Fix MainMenuGUI: semua create*Item() sekarang membaca material
    dari gui.yml via getGUIMaterial(), dengan fallback ke default jika
    config tidak ada
  → Fix playClickSound(): diganti dari switch hardcoded menjadi loop
    dinamis yang membaca slot dari config secara real-time
  → Fix: tax_item sekarang terdeteksi berdasarkan gui_slot (slot 22
    sesuai config), bukan slot 37 yang lama

  → president.max-consecutive-terms: 2 (limit masa jabatan berturut-turut)

- PlayerData.java:

  → Tambah method decrementPresidentCooldown() untuk mengurangi cooldown

- GovernmentManager.java:
  → canRunForPresident(): Sekarang memeriksa cooldown tracking dan max consecutive terms
  → getRemainingCooldownTerms(): Helper method untuk mendapatkan sisa cooldown player
  → setPresident(): Diperbarui untuk:
    * Set cooldown untuk president yang lengser jika mencapai max consecutive terms
    * Decrement cooldown untuk semua player saat term baru dimulai
    * Tracking consecutive terms yang lebih akurat

- ElectionManager.java:
  → Pesan error saat registrasi diperbarui untuk menampilkan sisa cooldown
    (contoh: "You must wait 1 more term(s) before running for president again.")

- Logika kerja:
  1. President yang mencapai max consecutive terms (2) akan mendapat cooldown
     sesuai config (default 1 term) saat lengser
  2. Setiap kali president baru dilantik, cooldown berkurang 1 untuk semua player
  3. Player dengan cooldown aktif tidak bisa mendaftar sebagai kandidat
