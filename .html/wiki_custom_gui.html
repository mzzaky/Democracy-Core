Overview

System Core

The updated DemocracyCore GUI system allows for unprecedented flexibility in configuring server menus through `gui.yml`. Administrators can now fully customize menu layouts, items, and interactions without modifying any Java code.

Key Features
  • Flexibility: Modify item functions without code changes.
  • Customization: Add or remove items easily.
  • Maintainability: Centralized configuration in `gui.yml`.
  • State Handling: Support for context-aware items (e.g., different icons if a President is active or not).
  • Adaptive Sizes: Configure the main menu size (9-54 slots).
  • Sound Effects: Play custom sounds on item clicks.

Configuration Structure

gui.yml

Each item in the GUI is defined in the configuration file with standard properties.

GUI Size Configuration

You can configure the size of the main menu inventory. Valid sizes are multiples of 9 (9, 18, 27, 36, 45, 54).

gui:
  main_menu:
    inventory_slot: 45  # Sets GUI size to 5 rows (45 slots)
    items: ...

Basic Item Format

gui:
  main_menu:
    items:
      unique_item_id:
        gui_slot: 10             # Slot in inventory (0-53)
        material: GOLDEN_HELMET  # Minecraft Material
        display_name: "§6Item Name"
        lore:
          - "Item description"
        click_sound: "entity.experience_orb.pickup" # Custom sound on click
        hide_attributes: ALL      # Hides enchantments/attributes
        on_click: "" # Action to trigger on click

Items with Multiple States

Some items can change their appearance based on server state (e.g., Active vs Inactive).

president_item:
  gui_slot: 10
  
  # State when president exists
  default:
    material: GOLDEN_HELMET
    display_name: "§6§l👑 PRESIDENT"
    on_click: ""
    
  # State when no president (election phase)
  no_president:
    material: BARRIER
    display_name: "§c§l👑 NO PRESIDENT"
    on_click: ""

Item Attributes & Flags

Hide Attributes

The hide_attributes feature allows administrators to hide specific information flags on GUI items, such as enchantments, attributes, durability, etc. This creates a cleaner and more professional look for the interface.

Usage Format

gui:
  main_menu:
    items:
      example_item:
        material: DIAMOND_SWORD
        display_name: "§aEpic Sword"
        hide_attributes: ALL  # <-- Add this option
        lore:
          - "§7Clean look without attributes"

Supported Values

Value Description
ALL Hides all item flags (Enchants, Attributes, etc.)
ENCHANTS Hides the list of enchantments
ATTRIBUTES Hides attribute modifiers (Damage, Speed, Armor)
UNBREAKABLE Hides the "Unbreakable" tag
DESTROYS Hides "Can Destroy" tag
PLACED_ON Hides "Can be Placed On" tag
POTION_EFFECTS Hides potion effects
DYE Hides dye information (Leather Armor)

Advanced Configuration

You can hide multiple specific flags by using a list format:

special_item:
  material: DIAMOND_SWORD
  hide_attributes:
    - ENCHANTS      # Hide enchantment list
    - ATTRIBUTES    # Hide attack damage/speed
Pro Tip
Use ALL for decorative weapons and armor to remove clutter like "Attack Damage" or "Armor" values. use ATTRIBUTES if you want to show enchantments but hide vanilla stats.

Player Info

Requirement Placeholders

Dynamic Conditions

These placeholders are used to display dynamic requirement values in item lore (e.g., for registering or voting).

Placeholder Description

{req_playtime} Required playtime (hours)
{req_balance} Required economy balance
{cost} Registration or action cost

GUI Actions

Interactions

The `on_click` property defines what happens when a player clicks an item. Actions are case-insensitive.

Navigation Actions (Open Menus)

Action Tag Description
<open_gui_main_menu> Opens Main Menu
<open_gui_government> Opens Government Dashboard
<open_gui_voting> Opens Election/Voting Menu
<open_gui_cabinet> Opens Cabinet Ministers Menu
<open_gui_treasury> Opens Treasury System Menu
<open_gui_executive_orders> Opens Executive Orders Menu
<open_gui_recall> Opens Recall/Impeachment System
<open_gui_player_stats> Opens Player Statistics Profile
<open_gui_history> Opens Presidential History Log
<open_gui_leaderboard> Opens Server Leaderboards
<open_gui_help> Opens Help & Guide Menu

Special Actions (Functional)

Action Tag Effect
<action_register_candidate> Registers player as presidential candidate
<action_rate_president> Prompts to rate the current president
<action_endorse_candidate> Prompts to endorse a candidate
<action_donate_treasury> Prompts to donate to treasury
<action_join_arena> Joins the Presidential Arena (if active)
<action_arena_info> Displays Arena status and stats
<action_refresh_gui> Reloads/Refreshes the current menu
<action_close_inventory> Closes the current menu immediately
<action_console_command> Executes a console command (Requires `value` field)

Implementation Guide

Tutorial

How to Add a New Item

  1. Open src/main/resources/gui.yml.
  2. Navigate to the items section of the desired menu.
  3. Add a new unique key for your item.
  4. Define the material, slot, name, and action.
Example: Adding a Discord Link
discord_link:
  gui_slot: 45
  material: BLUE_WOOL
  display_name: "§9Join Discord"
  lore:
    - "§7Click to get invite link"
  on_click: "" # Or a custom command action

Troubleshooting

  • Item not appearing? Check if `gui_slot` overlaps with another item.
  • Action not working? Ensure the `on_click` tag is wrapped in angle brackets < >.
  • YAML Errors: Use a YAML validator to ensure correct indentation.
