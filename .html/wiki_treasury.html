Overview

Economy Module

This document explains the technical details of the **National Treasury** system in DemocracyCore. This system manages the circulation of money, state revenue, and government spending to finance policies.

Key Functionality
The Treasury is an economic entity separate from any player's personal account. The state has its own balance used for:
  • Financing president's Executive Orders.
  • Funding ministers' Cabinet Decisions.
  • Paying officials' salaries (President & Ministers).
  • Providing Voter Rewards.

If the state treasury is empty, the government cannot function effectively (economic collapse).

Income Sources

Revenue

1. Transaction Tax

Code: TAX_INCOME

Percentage of every transaction goes to treasury.

  • Default: 5% of transaction value.
  • Config: treasury.transaction-tax

2. Fines & Seizures

Code: FINE_INCOME

Money from failed petitions and penalties.

  • Failed Recall deposits.
  • Loser candidate registration fees (if configured).

3. Donations

Code: DONATION

Voluntary contributions from citizens.

  • Command: /dc treasury donate <amount>

4. Initial Fund

Code: TERM_START_FUND

Capital injection at start of new term (admin controlled).

Government Expenses

1. Government Programs

  • Executive Orders: High cost to activate global policies (e.g., Golden Age).
  • Cabinet Decisions: Operational costs for ministries (e.g., Festivals, Subsidies).

2. Salaries (CABINET_SALARY)

Daily wages paid to officials automatically at specific times.

Role Default Salary
President $5,000 / day
Minister $2,500 / day

3. Public Rewards

  • Voter Reward: Cash prizes for voting citizens.
  • Stimulus: Direct cash assistance (BLT) from Treasury Minister.
  • Deposit Refund: Campaign deposit return for election winners.

Transaction Management

The system records every inflow and outflow transparently.

  • Logging: Every deposit/withdraw is recorded with: Type, Amount, Description, Actor (UUID), and Timestamp.
  • Log Limit: Stores last 1000 transactions for audit.
  • Audit: Players can view history via GUI or commands.

Commands

Command Function Permission
/dc treasury View financial status (Balance, Income, Expense, Logs). Public
/dc treasury donate <amount> Donate money to state treasury. Public
/dc admin addtreasury <amount> Instantly add money to treasury (Cheat). Admin

Configuration

config.yml

Economic settings are located in config.yml under the treasury section.

treasury:
  # Transaction tax (0.05 = 5%)
  transaction-tax: 0.05
  
  # Daily salaries
  salaries:
    president: 5000
    minister: 2500
