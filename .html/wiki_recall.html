Overview

Core Module

This document explains the technical details of the **Recall System** (Impeachment) module in DemocracyCore. This system empowers the people to remove a president who is deemed incompetent or abusing power before their term ends.

Key Functionality
The Recall System is designed as a mechanism for checks and balances. This process is not easy to carry out, requiring financial support and a significant majority vote to prevent abuse of features by minority groups.

The impeachment process is divided into two main phases:

  1. Petition Phase (Collecting): Collecting signatures of support.
  2. Voting Phase (Recall Vote): Referendum on the president's fate.

Recall Process Flow

4 Steps

1. Initiation

A player starts a petition.

  • Requirements: President active, not self, no active petition, cooldown over.
  • Command: /dc recall start
  • Cost: $50,000 Deposit

2. Signature Collection

Duration: 7 Days

Players sign petition to support recall.

  • Cost to Sign: $50,000 (Refundable if success)
  • Target: 30% of weekly active players

3. Voting Phase

Duration: 3 Days

If signatures met, referendum begins.

  • Options: REMOVE (Red) vs KEEP (Green)
  • Command: /dc recall vote <remove/keep>

4. Final Decision

Result determined by Super-Majority (60%).

See "Results & Consequences" for details.

Results & Consequences

✅ Success Scenario (President Removed)
Occurs if REMOVE votes reach super-majority (Default: 60%).
  • President Dismissed: President immediately loses office.
  • Full Refund: All deposits returned 100%.
  • Emergency Election: Triggered automatically.
  • Sound Effect: Global Wither Death sound played.
❌ Failed Scenario (President Stays)
Occurs if REMOVE votes are below threshold.
  • President Safe: Term continues as normal.
  • Deposit Forfeited: All deposits seized to National Treasury.
  • Cooldown: System enters cooldown (15 Days).

Security & Configuration

config.yml

To prevent spam and abuse, the system implements:

  • High Deposit: Prevents frivolous petitions.
  • Dynamic Requirement: Based on active players in last 7 days.
  • Super-Majority: Requires 60% agreement, not just simple majority.

Configuration

recall:
  # Cooldown time before new petition after failure
  cooldown-days: 15
  
  # Duration of signature collection
  collection-days: 7
  
  # Duration of recall voting
  voting-days: 3
  
  # Security deposit for signers
  signature-deposit: 50000
  
  # Percentage of active players needed to trigger vote
  required-signature-percentage: 30
  
  # Percentage of 'REMOVE' votes needed for success
  required-percentage: 60

Admin Commands

Admins can intervene in the process:

  • /dc admin cancelrecall <reason>: Cancel active petition and refund everyone.
  • /dc admin removepresident: Force remove president without voting.
